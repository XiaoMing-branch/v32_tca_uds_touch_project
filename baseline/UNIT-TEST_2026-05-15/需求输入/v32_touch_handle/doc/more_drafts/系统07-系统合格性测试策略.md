## 1. 引言

### 1.1 目的
本文档定义门把手触摸功能的**系统合格性测试策略**，旨在通过全面的黑盒测试验证产品是否满足《系统需求分析分解结果》中的所有需求，确保其在交付前具备完整的功能、性能与可靠性表现。

> **说明**：本项目为非 ASIL 舒适性功能，合格性测试聚焦于用户场景、接口合规性与环境适应性，不包含 ISO 26262 功能安全相关验证。

### 1.2 范围
- **测试对象**：门把手总成（含硬件、固件）+ BCM 仿真器
- **覆盖内容**：
  - 端到端功能流程（触摸 → 开锁信号）
  - GPIO 与 LIN 接口行为
  - 低功耗运行与唤醒
  - 环境适应性（高低温、振动、防水）
  - 可靠性与耐久性
- **不包含**：
  - 单元模块内部逻辑（已在软件/系统集成测试中覆盖）
  - HIL 或实车动态测试（由整车级测试完成）

### 1.3 参考文档
- 《系统需求分析分解结果》SYS-REQ-BREAKDOWN-01
- 《系统集成测试报告》STR-SYS-INT-V1.0

## 2. 测试目标

| 目标类别 | 具体目标 |
|--------|--------|
| **需求符合性** | 100% 覆盖 SRS 分解结果中的 37 项需求 |
| **场景完整性** | 覆盖正常使用、维护升级、异常干扰等真实用户场景 |
| **鲁棒性验证** | 验证在电源波动、电磁干扰、物理应力下的稳定性 |
| **可交付性** | 满足发布基线要求，无高优先级缺陷 |

---

## 3. 测试方法与级别

### 3.1 测试方法
采用**基于风险的黑盒测试（Risk-Based Black-Box Testing）**：

| 方法 | 说明 |
|------|------|
| **需求驱动测试** | 每项 SRS 子需求映射至少 1 个合格性用例 |
| **场景化测试** | 模拟“用户接近→触摸→解锁→睡眠”完整路径 |
| **边界值分析** | 测试电压极限（9V/16V）、温度极限（-40°C/+85°C） |
| **错误注入** | 制造无效 LIN 帧、断电、传感器失效等异常 |

### 3.2 测试级别
本测试为**系统级合格性测试（System Qualification Test, SQT）**，在完整软硬件系统上执行，是产品发布的最后一道质量关卡。

## 5. 测试内容分类

### 5.1 功能合格性
- 触摸检测准确性（戴手套、湿手等）
- 开锁信号输出时序与电平
- 防误触机制有效性（水、振动、泡沫）

### 5.2 性能合格性
- 触摸响应延迟 ≤100ms
- 休眠电流 ≤200μA
- OTA 更新时间 ≤90s（128KB）

### 5.3 接口合格性
- GPIO 信号兼容性（3.3V/5V）
- LIN 协议一致性（DID 读取、OTA 流程）
- NVM 数据持久性（断电恢复）

### 5.4 环境与可靠性
- -40°C 至 +85°C 下功能正常
- IP67 防水测试（1m 水深 30 分钟）
- EMC 抗扰度（10V/m 射频干扰）
- 20万次耐久测试（触摸寿命）

### 5.5 用户场景覆盖
| 场景类型 | 示例 |
|--------|------|
| 正常使用 | 日常触摸解锁 |
| 维护场景 | OTA 升级、诊断读取 |
| 异常操作 | 冲洗、结冰、戴厚手套 |
| 极端环境 | 高低温启动、电源波动 |

## 6. 通过/失败准则

### 6.1 通过准则（必须全部满足）
- **100% SRS 需求覆盖**：所有 37 项子需求均有对应测试用例并通过；
- **关键功能零缺陷**：
  - 开锁信号输出正确
  - 无误触发
  - LIN 诊断可读
- **性能达标**：响应时间、功耗、OTA 时间符合规格；
- **无高优先级缺陷**：无阻塞用户核心功能的问题。

### 6.2 缺陷分类与处理
| 优先级 | 定义 | 处理要求 |
|--------|------|--------|
| **高（Critical）** | 导致无法开锁、误触发、OTA 永久失效 | 必须修复，否则拒绝发布 |
| **中（Major）** | 诊断读错、计数偏差、LED 异常 | 需评审，可带缺陷发布（记录偏差） |
| **低（Minor）** | 日志格式错误、非关键时序轻微超差 | 可接受，记录于已知问题列表 |

| 阶段 | 活动 | 周期 | 交付物 |

## 9. 风险与缓解

| 风险 | 缓解措施 |
|------|--------|
| 样件供应不足 | 提前锁定 3 套测试专用样件，禁止挪用 |
| 环境设备冲突 | 提前预约高低温箱和振动台，分时使用 |
| 需求理解偏差 | 所有用例需经系统需求负责人（粟工）签字确认 |
| 第三方测试延迟 | 提前委托实验室排期，预留缓冲时间 |
