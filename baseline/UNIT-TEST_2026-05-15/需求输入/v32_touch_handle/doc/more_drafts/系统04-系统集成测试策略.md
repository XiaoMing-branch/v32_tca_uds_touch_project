**文档编号：STP-SYS-INT-V1.0**
**版本：V1.0**
**发布日期：2025-11-11**
**编制人：钟工**

---

## 1. 引言

### 1.1 目的
本文档定义门把手触摸功能的**系统集成测试策略**，确保硬件、软件、外部控制器之间的接口与协同行为满足《系统需求分析分解结果》中的全部要求。重点验证“用户触摸 → 开锁信号输出 → 车身响应”端到端流程，以及LIN诊断/OTA功能在整车环境下的正确性。

### 1.2 范围
- **测试对象**：门把手总成（含MCU、电容传感器、外壳） + 车身控制器（BCM）仿真器 + LIN通信设备
- **覆盖内容**：
  - 触摸检测与开锁信号输出
  - GPIO硬线接口时序与电平兼容性
  - LIN通信（诊断DID读取、OTA更新）
  - 系统级低功耗行为（运行↔睡眠）
  - 异常处理与恢复能力
- **不包含**：
  - 单元模块内部逻辑（已在软件集成测试中完成）
  - 硬件可靠性试验（如EMC、IP67、寿命测试）

## 2. 测试目标

| 目标类别 | 具体目标 |
|--------|--------|
| **功能集成** | 验证从用户触摸到BCM接收开锁信号的完整链路 |
| **接口一致性** | 验证GPIO信号、LIN协议符合定义 |
| **状态同步** | 验证门把手与BCM在电源模式、故障状态上的一致性 |
| **鲁棒性** | 验证异常输入、干扰环境下系统的健壮性 |

---

## 3. 测试方法与级别

### 3.1 测试方法
采用**黑盒测试 + 场景驱动**为主：

| 方法 | 说明 |
|------|------|
| **端到端场景测试** | 模拟真实用户操作路径（如“接近→触摸→解锁→睡眠”） |
| **接口协议测试** | 使用LIN工具验证DID读取、OTA流程 |
| **边界值测试** | 测试电压极限、温度极限下的功能表现 |
| **错误注入测试** | 人为制造无效帧、断电信号等异常 |

### 3.2 测试级别
本测试为**系统级集成测试（System Integration Test, SIT）**，运行于完整软硬件原型之上，包含以下层级交互：
- 门把手 ECU ↔ 车身控制器（BCM）
- 门把手 ECU ↔ 诊断设备 / OTA服务器
- 用户 ↔ 产品物理界面

## 5. 测试内容分类

### 5.1 核心功能集成
验证用户触摸 → 开锁信号 → BCM 响应全流程。

### 5.2 接口集成
验证 GPIO 和 LIN 接口的电气特性与协议一致性。

### 5.3 电源与低功耗集成
验证门把手与 BCM 在休眠/唤醒过程中的协同行为。

### 5.4 异常与恢复集成
验证通信错误、电源波动、误触发抑制等场景下的系统稳定性。

### 5.5 环境适应性集成
验证高低温、振动、湿热环境下系统功能完整性。

## 6. 关键测试场景设计

| 场景ID | 场景名称 | 描述 | 预期结果 | 验证方式 |
|--------|--------|------|--------|--------|
| **SCN-SYS-01** | 正常开锁流程 | 用户触摸 → 输出信号 → BCM 解锁车门 | BCM 在 ≤150ms 内执行解锁动作 | 示波器 + BCM 日志 |
| **SCN-SYS-02** | 防重复触发 | 连续两次快速触摸（间隔 <500ms） | 仅触发一次开锁信号 | 示波器观测脉冲数 |
| **SCN-SYS-03** | 自动睡眠与唤醒 | 无操作 5 秒后进入睡眠 → 触摸唤醒 | 睡眠电流 ≤200μA；唤醒后正常响应 | 电流表 + 示波器 |
| **SCN-SYS-04** | LIN 诊断读取 | 通过 LIN 请求 DID 0xF180（版本）、0xF190（计数） | 返回正确数据，BCM 可解析 | LIN 工具抓包 |
| **SCN-SYS-05** | OTA 固件更新 | 发起 OTA → 下载 → 提交 → 重启生效 | 新固件版本被激活，功能正常 | 版本对比 + 功能回归 |
| **SCN-SYS-06** | 低温工作（-40°C） | 在 -40°C 下执行 10 次触摸操作 | 所有操作均成功识别并输出信号 | 高低温箱 + 示波器 |
| **SCN-SYS-07** | 电源波动（9V–16V） | 输入电压在 9V/12V/16V 间切换 | 功能稳定，无复位或误触发 | 可编程电源 + 观察 |
| **SCN-SYS-08** | LIN 错误帧处理 | 发送 CRC 错误、非法 SID 的 LIN 帧 | 忽略错误帧，不影响主功能 | LIN 工具 + 系统日志 |
| **SCN-SYS-09** | 断电恢复 | 上电 → 触摸 → 断电 → 重新上电 → 读计数 | NVM 中触摸计数保持不变 | LIN 读 DID 对比 |
| **SCN-SYS-10** | 抗水误触 | 喷洒水流模拟洗车场景 | 不产生误触发，仅真实触摸响应 | 视频记录 + 信号监测 |

## 7. 通过/失败准则

### 7.1 通过准则（必须全部满足）
- 所有 **SRS 分解需求**均有对应测试用例且通过；
- **核心功能零缺陷**：
  - 开锁信号输出及时准确
  - 防重复机制有效
  - LIN 诊断可读
- **性能达标**：
  - 触摸→信号延迟 ≤100ms
  - 睡眠电流 ≤200μA
  - OTA 时间 ≤90s（128KB）
- **无高优先级缺陷**（阻塞发布类问题）。

### 7.2 缺陷分类标准
| 优先级 | 定义 | 处理要求 |
|--------|------|--------|
| **高（Critical）** | 导致无法开锁、误触发、OTA 永久失效 | 必须修复，否则拒绝发布 |
| **中（Major）** | 诊断读错、计数偏差、LED 异常 | 需评审，可带缺陷发布（记录偏差） |
| **低（Minor）** | 日志格式错误、非关键时序轻微超差 | 可接受，记录于已知问题列表 |
