## 2. 产品概述

### 2.1 产品定义
智能电容触摸门把手是一款集成电容感应技术的汽车门把手，用户通过轻触即可触发本地开锁请求。系统采用**硬线 GPIO 输出开锁信号**，**LIN 接口用于 OTA 固件更新和诊断服务**，实现功能解耦与高响应可靠性。

### 2.2 产品定位
- **目标市场**：中高端乘用车
- **产品类型**：电子门锁触发装置（非执行器）
- **技术特点**：电容感应、低功耗运行、防误触设计

### 2.3 产品边界
- **包含**：电容传感器、控制电路、外壳、连接线束、开锁信号输出逻辑
- **不包含**：车门锁体、车身控制器（BCM）、钥匙系统、电机驱动模块

> **说明**：本产品为“信号发生器”，不直接控制锁体动作，仅提供标准开锁触发信号供上级控制器使用。

### 6.4 开锁触发信号输出接口详细规格

| 项目 | 规格 |
|------|------|
| **输出类型** | 推挽输出（默认高电平） |
| **信号功能** | 低电平 = 请求解锁 |
| **脉冲宽度** | (320ms~2000ms)±20ms |
| **默认状态** | 高电平（无效） |
| **重复触发抑制** | 在 320ms 脉冲期间忽略新的触摸事件 |
| **驱动能力** | ≥10mA @ 12V |
| **响应时间** | 从有效触摸到下降沿 ≤100ms |
| **用途** | 向车身控制器（BCM）发送本地开锁请求 |

## 8. 用户场景

### 8.1 正常使用场景
1. 用户接近车门并触摸门把手感应区
2. 系统检测到有效触摸（排除雨水、振动等干扰）
3. 输出一个 **320ms 低电平开锁信号，最长可达2S**
4. 车身控制器接收信号后执行解锁动作
5. 用户开门

### 8.2 维护与升级场景
1. 维修人员连接诊断设备至 LIN 接口
2. 通过 LIN 执行 OTA 软件刷写
3. 读取设备诊断信息（如软件版本、故障码）
4. 获取当前运行状态

### 8.3 异常操作场景
1. 用户戴手套触摸 → 系统识别为有效触摸 → 正常输出信号
2. 雨天环境触摸 → 系统过滤雨水干扰 → 仅在真实手部接触时触发
3. 连续快速触摸（<320ms间隔）→ 仅响应第一次 → 避免误操作

### 8.4 低功耗场景
1. 车辆熄火后，门把手进入休眠模式（电流 ≤200μA）
2. 用户触摸门把手 → 外部中断唤醒系统
3. 检测到有效触摸 → 输出 320ms 开锁信号
4. 若无进一步操作持续 5 秒 → 自动返回休眠

## 9. 验证计划

### 9.1 验证方法
- **单元测试**：验证单个功能模块（如触摸检测、信号输出）
- **集成测试**：验证“触摸→信号输出”端到端流程
- **环境测试**：验证高低温、湿度、振动下的稳定性
- **可靠性测试**：模拟长期使用（20万次触摸）
- **接口测试**：验证 LIN 通信与 GPIO 信号时序

### 9.2 验证标准
- 所有功能需求必须通过测试用例验证
- 性能指标满足规格要求（如响应时间、功耗）
- 环境与可靠性测试通过行业标准
- LIN 与 GPIO 接口行为符合定义，无交叉干扰
