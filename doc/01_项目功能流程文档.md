# V32 车规级电容式外把手 — 项目功能流程文档

> **芯片平台**: Tinychip TCPL01X / TCPL03X (TCAE10X)  
> **通信总线**: LIN 2.x / J2602 (Slave 模式，波特率 19200bps)  
> **诊断协议**: UDS on LIN (ISO 14229-1 / LIN 诊断规范)  
> **OTA 机制**: Bootloader + App 双区架构，通过 UDS 诊断服务实现固件下载与激活  
> **版本**: V1.0  
> **日期**: 2026-05-25

---

## 2. 上电启动流程

### 2.1 启动序列

```
Power On Reset
      │
      ▼
  [Cortex-M0 Reset Vector]
      │
      ├──► Bootloader 入口 (__main / Reset_Handler)
      │       │
      │       ├── 1. 系统初始化
      │       │     ├── 时钟配置 (CRG)
      │       │     ├── 中断向量表重定位
      │       │     ├── SysTick 定时器初始化 (1ms 时基)
      │       │     └── GPIO / PWM / ADC 初始化
      │       │
      │       ├── 2. dfu_manager_init()
      │       │     ├── 从 Flash 读取 NAD 配置
      │       │     ├── LIN 控制器初始化 (Slave, 19200bps)
      │       │     └── 初始化 DFU 上下文
      │       │
      │       ├── 3. main_loops() 主循环
      │       │     │
      │       │     ├── [循环] lin_diag_service_handle()
      │       │     │     └── 持续监听 LIN 诊断请求
      │       │     │
      │       │     ├── [约 42ms 后]
      │       │     │     ├── 检查上次 DFU 信息
      │       │     │     ├── 成功 → BOOT_STATE_USER_APP
      │       │     │     └── 失败 → BOOT_STATE_UPGRADE
      │       │     │
      │       │     └── [BOOT_STATE_USER_APP]
      │       │           └── JumpToApp()
      │       │                 ├── 关闭中断 (NVIC)
      │       │                 ├── 反初始化 LIN
      │       │                 ├── 设置 MSP（应用栈指针）
      │       │                 └── 跳转到 App 入口地址
      │       │
      │       └── [BOOT_STATE_UPGRADE]
      │             └── 停留在 Bootloader，等待 OTA 指令
      │
      └──► App 应用程序入口
              │
              ├── 1. 系统初始化
              │     ├── 外设初始化 (GPIO/ADC/PWM/LIN)
              │     ├── 中间件初始化
              │     │     ├── LIN 协议栈初始化
              │     │     ├── 存储模块初始化
              │     │     └── 触摸检测初始化
              │     └── 参数加载（从 Flash）
              │
              ├── 2. 进入主循环 (lin_task)
              │     ├── 10ms 任务: LIN 报文收发
              │     ├── 10ms 任务: 电容检测处理
              │     ├── 100ms 任务: LED 状态更新
              │     └── 100ms 任务: 诊断服务处理
              │
              └── 3. 正常运行模式
```

### 2.2 Bootloader 启动决策逻辑

```
上电复位
    │
    ▼
Bootloader 初始化
    │
    ▼
进入 main_loops()
    │
    ├── 每次循环：lin_diag_service_handle()
    │   监听 LIN UDS 请求（允许进入编程会话 OTA）
    │
    ├── 累计 ~50 次循环 (≈42ms)
    │   达到计时条件后查询 last_dfu_info
    │
    ├── DFU 信息有效 (magic=0xDEADBEEF, reason=SUCCESS)
    │    → BOOT_STATE_USER_APP
    │    → 跳转 App
    │
    └── DFU 信息无效
         → BOOT_STATE_UPGRADE
         → 停留在 Bootloader，等待 OTA
```

**关键设计点**:
- Bootloader 上电后先等待 ~42ms 监听诊断请求，确保可以立即响应 OTA 指令
- 如果在等待期间收到 `0x10 0x02` (编程会话)，则进入升级模式
- 此等待窗口使 BCM 能在极短时间内触发 OTA

## 3. 电容检测流程

电容检测基于 Tinychip 自容/互容检测方案，由 `touch/` 中间件层实现。

### 3.1 架构

```
┌─────────────────────────────────────────────┐
│           Touch Application Layer            │
│  si_touch_port.c/h (Sensor 配置 & 数据处理)  │
├─────────────────────────────────────────────┤
│           Touch HAL (硬件抽象层)              │
│  touch_haldispatch.c/h (调度器，状态机)      │
│  touch_halnode.c/h (节点扫描控制)            │
├─────────────────────────────────────────────┤
│           Tool / Utility                     │
│  touch_tool.c/h (滤波、校准、阈值)           │
└─────────────────────────────────────────────┘
```

### 3.2 检测流程

```
System Tick (1ms 时基)
      │
      ▼
touch_haldispatch 调度器
      │
      ├── 自容扫描 (Self-Capacitance)
      │     ├── 电极充电
      │     ├── 测量充电时间/频率变化
      │     └── 原始数据采集
      │
      ├── 互容扫描 (Mutual-Capacitance)
      │     ├── 驱动电极 TX 发送激励信号
      │     ├── 接收电极 RX 测量耦合电容
      │     └── 原始数据采集
      │
      └── 数据处理 (touch_tool)
            ├── 数字滤波 (均值/中值/低通)
            ├── 基线跟踪 (Baseline Tracking)
            ├── 环境补偿 (温度/湿度)
            └── 触摸判决 (阈值比较 + 去抖动)
```

### 3.3 触摸状态机

```
           ┌──────────────────┐
           │   IDLE (空闲)     │
           └────────┬─────────┘
                    │ 检测值 > 阈值
                    ▼
           ┌──────────────────┐
           │  PRESS_DETECT    │ ─── 去抖动计时
           └────────┬─────────┘
                    │ 去抖动确认 (N 个周期)
                    ▼
           ┌──────────────────┐
           │   PRESS (按下)    │ ─── 触发动作 (解锁/闭锁请求)
           └────────┬─────────┘
                    │ 检测值 < 阈值 (释放)
                    ▼
           ┌──────────────────┐
           │  RELEASE_DETECT  │ ─── 去抖动计时
           └────────┬─────────┘
                    │ 确认释放
                    ▼
           ┌──────────────────┐
           │   IDLE (空闲)     │
           └──────────────────┘
```

```
├────────────────────────────────────────────────────┤
│   LIN Transport Layer (TL)                         │
│   (linlib - 单帧/多帧传输，流控制)                  │
├────────────────────────────────────────────────────┤
│   LIN Protocol Layer (API)                         │
│   (lin.h / linlib.h - NAD, PID, 校验)              │
├────────────────────────────────────────────────────┤
│   LIN Driver Layer (LLD)                           │
│   (lin_process.c - 中断处理，状态机，收/发)          │
├────────────────────────────────────────────────────┤
│   PAL (Platform Abstraction Layer)                 │
│   (pal_lin_comm.c - SCI 寄存器操作)                 │
├────────────────────────────────────────────────────┤
│   Hardware (LIN SCI Transceiver)                   │
└────────────────────────────────────────────────────┘
```

### 4.2 LIN 从节点状态机

```
        ┌──────────────┐
        │   UNINIT     │  ← 上电/复位初始状态
        └──────┬───────┘
               │ lin_lld_sci_init()
               ▼
        ┌──────────────┐
        │    IDLE      │  ← 等待总线活动
        └──────┬───────┘
               │ Break 检测到 (LIN_INT_BREAK_DET_FLAG)
               ▼
        ┌──────────────┐
        │  RECV_SYN    │  ← 接收同步字节 (0x55)
        └──────┬───────┘
               │ Sync 检测到 (LIN_INT_SYNC_DET_FLAG)
               ▼
        ┌──────────────┐
        │  RECV_PID    │  ← 接收PID (保护标识符)
        └──────┬───────┘
               │ PID OK → 检查是否是本节点 NAD
               ▼
        ┌──────────────────┐
        │  RECV_DATA       │  ← 从机响应/接收数据
        │  or SEND_DATA    │  ← 从机发送数据
        └──────────────────┘
               │ 传输完成 (校验和验证)
               ▼
        ┌──────────────────┐
        │ PROC_CALLBACK    │  ← 触发上层回调
        └──────┬───────────┘
               │
               ▼
        ┌──────────────┐
        │    IDLE      │  ← 回到空闲，等待下一帧
        └──────────────┘
        
        ┌──────────────────┐
        │ LIN_SLEEP_MODE   │  ← 总线空闲超时进入睡眠
        └──────────────────┘
```

### 4.3 报文接收流程 (中断驱动)

```
LIN Break 到达
    │
    ▼
ISR: LIN_INT_BREAK_DET_FLAG
    │  → state = RECV_SYN，启动帧超时计数器
    │
    ▼
ISR: LIN_INT_SYNC_DET_FLAG
    │  → 自动波特率检测 (pal_lin_autobaudrate_check)
    │  → state = RECV_PID
    │
    ▼
ISR: LIN_INT_RX_PID_DONE_FLAG
    │  → 读取 PID 字节
    │  → 校验 PID 奇偶性 (lin_process_parity)
    │  → 匹配成功后：CALLBACK_HANDLER(LIN_LLD_PID_OK)
    │  → 设置响应超时计数器
    │
    ▼
ISR: LIN_INT_RX_1BYTE_FLAG (逐字节)
    │  → 逐字节接收数据 (置入 response_buffer)
    │  → 当 cnt_byte == 报文长度时
    │      → 校验 checksum (lin_checksum)
    │      → 成功: CALLBACK_HANDLER(LIN_LLD_RX_COMPLETED)
    │      → 失败: CALLBACK_HANDLER(LIN_LLD_CHECKSUM_ERR)
    │
    ▼
回调触发 → LIN 协议层解析 → 诊断分发
```

### 4.4 报文发送流程

```
上层调用 ld_send_message() / lin_uds_send()
    │
    ▼
准备响应数据 (SID+0x40 正响应 或 0x7F 负响应)
    │
    ▼
响应置于 response_buffer
    │
    ▼
LIN Master 发送下一帧的 PID（匹配 NAD 的帧）
    │
    ▼
ISR: PID OK → 状态为发送节点
    │  → pal_lin_tx_response(pid, buffer, length)
    │  → state = SEND_DATA
    │
    ▼
ISR: LIN_INT_TX_DONE_FLAG
    │  → state = PROC_CALLBACK
    │  → CALLBACK_HANDLER(LIN_LLD_TX_COMPLETED)
    │  → 回到 IDLE
    │
    ▼
发送完成
```

### 4.5 App 任务调度 (lin_task)

```
lin_task 主循环 (由 OS 定时调用)
    │
    ├── [10ms] lin_process_10ms()
    │     ├── lin_diag_service_handle() → UDS 诊断服务处理
    │     └── LIN 协议栈时基更新
    │
    └── [100ms] lin_process_100ms()
          ├── 触摸状态处理
          └── LED 状态更新
```

## 5. 正常工作模式 / 升级模式切换逻辑

### 5.1 模式定义

| 运行模式 | 执行固件 | 描述 |
|---------|---------|------|
| **正常运行模式** | App | 电容检测、LED 控制、LIN 通信、正常诊断 |
| **Bootloader 升级模式** | Boot | 等待 UDS 编程会话，擦写 Flash，固件下载 |
| **过渡态 (待跳转)** | Boot | 等待 42ms 后自动跳转至 App |

### 5.2 正常 → 升级切换

```
正常运行模式 (App)
    │
    ├── 收到 UDS $10 $02 (DiagnosticSessionControl: ProgrammingSession)
    │     └── App 内部触发 NVIC_SystemReset()
    │
    ▼
系统复位
    │
    ▼
Bootloader 启动
    │
    ├── 初始化 LIN
    │
    ├── 进入 main_loops()
    │     │
    │     ├── 持续监听 UDS 请求
    │     │
    │     └── [42ms 等待窗口内]
    │           │
    │           └── 收到 $10 $02 (编程会话)?
    │                 └── YES → BOOT_STATE_UPGRADE
    │                       ├── 点亮 LED 指示
    │                       └── 进入 OTA 流程
    │
    └── 42ms 内未收到编程请求
          └── BOOT_STATE_USER_APP → JumpToApp()
```

### 5.3 升级 → 正常切换 (OTAF 成功后)

```
Bootloader 升级模式
    │
    ├── UDS OTA 流程 (编程会话)
    │     ├── $10 $02 编程会话控制
    │     ├── $27 安全访问 (解锁)
    │     ├── $2E 固件信息同步
    │     ├── $31 $02 $FF00 擦除 Flash
    │     ├── $34 $01 请求下载
    │     ├── $36 传输数据 (多包)
    │     ├── $37 请求传输结束
    │     └── $31 $01 $0202 校验 CRC → 写入 DFU 信息
    │
    ├── 写入 last_dfu_info (magic=0xDEADBEEF, reason=SUCCESS)
    │
    └── 系统复位 / 跳转
          │
          ▼
Bootloader 启动
    │
    ├── last_dfu_info 有效 → BOOT_STATE_USER_APP
    │
    └── JumpToApp() → 正常运行模式 (新固件)
```

### 5.4 升级失败处理

```
OTA 过程中断/超时 (3 秒无 UDS 请求)
    │
    ▼
dfu_process_exit(DFU_MSG_TIMEOUT)
    │
    ├── 清除所有上下文
    ├── dfu_ctx.boot_state = BOOT_STATE_UPGRADE
    └── 停留在 Bootloader，等待下一次 OTA 尝试

Flash 擦写失败
    │
    ▼
dfu_process_exit(DFU_MSG_ERASE_ERROR)
    │
    └── Bootloader 等待后续指令
        (不写入 DFU 信息，下次上电仍进入 BOOT)
```

## 6. 关键数据结构

### 6.1 DFU 上下文 (boot 侧核心结构)

```c
typedef struct {
    uint32_t write_addr;          // 当前写入 Flash 地址
    uint32_t write_length;        // 已写入字节数
    uint32_t write_index;         // 写入数据包序号
    uint32_t write_crc;           // 实时 CRC32 校验值
    uint32_t receive_length;      // 已接收字节数
    uint32_t recevice_index;      // 已接收数据包序号
    uint32_t uds_timeout;         // UDS 超时计数 (ms)
    uint8_t  op_code;             // 当前操作码 (DFU_CMD_xxx)
    uint8_t  resp_value;          // 响应值/错误码
    uint8_t  program_flag;        // 编程标志
    uint8_t  boot_state;          // Boot 状态 (IDLE/USER_APP/UPGRADE)
    lin_baudrate_config_t lin_config;   // LIN 波特率配置
    last_dfu_info_t dfu_info;           // 上次 DFU 信息
    queue_list_t queue_list;            // 数据包双缓冲队列
} dfu_manager_context_t;
```

### 6.2 UDS 服务描述表

```c
typedef struct {
    uint8_t sid;            // 服务标识符
    uint8_t sessionMode;    // 允许的会话模式 (位掩码)
    uint8_t requsetMode;    // 请求模式 (物理/功能寻址)
    uint8_t securityLevel;  // 所需安全级别
} ServiceUDS_TypeDef;
```
